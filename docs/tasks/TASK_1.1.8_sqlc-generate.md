## 1. Summary

Run `sqlc generate` after task 1.1.7 is complete, commit the resulting Go files under `internal/platform/db/sqlc/`, and add a CI step that verifies the committed generated code is always in sync with the query files. Any drift between `.sql` source and committed Go output fails the pipeline.

---

## 2. Context & Motivation

`sqlc` is a code-generation tool, not a runtime library. The generated Go files must be committed so that the repo is self-contained and does not require `sqlc` to be installed in every consumer environment. The CI sync-check (already present in `ci.yml`) ensures no query file is changed without regenerating — preventing silent divergence between the SQL source of truth and the Go code that teams actually run.

- Roadmap row: `docs/ROADMAP.md` › Phase 1 › 1.1.8
- CI reference: `.github/workflows/ci.yml` — "Check sqlc generation is up to date" step
